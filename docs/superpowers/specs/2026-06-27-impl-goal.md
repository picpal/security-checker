# 구현 Goal — JVM 보안 점검 하이브리드 도구 (전체 M0~M5)

> 실행: **Claude Code 새 세션.** 긴 자율 구현이므로 **컨텍스트 관리가 성패를 가른다.**
> 아래 "세션 운영 규칙"을 가장 먼저 셋업하고 시작할 것.

---

## 목표

JVM(Java/Kotlin) 프로젝트용 **보안 점검 하이브리드 CLI 도구**를 spec대로 **M0부터 M5까지 순차 구현**한다. 결정적 오케스트레이터가 오픈소스 스캐너를 병렬 실행·정규화하고, 도달성 분석으로 노이즈를 걷어낸 뒤, Claude가 증거를 해석해 보고서를 낸다.

## 필독 (시작 전 + 매 마일스톤 시작 시)

이 spec이 **최우선 기준**이다. 이 goal과 충돌하면 spec을 따른다:
```
docs/superpowers/specs/2026-06-26-jvm-security-scanner-design.md
```

## 세션 운영 규칙 (compact 대응 — M0보다 먼저 셋업)

이 작업은 길어서 context가 차면 compact(요약)된다. compact 후에도 목표·제약·진행을 잃지 않도록 **첫 3가지를 즉시 셋업**한다:

1. **`CLAUDE.md` 생성** (Claude Code가 매 턴 자동 로드 → compact에 가장 강함). 내용:
   - 아래 "핵심 원칙 6개"를 그대로 복사
   - 경로 3개: 이 goal, spec, `PROGRESS.md`
   - `현재 마일스톤: Mx` 한 줄 (진행하며 갱신)
   - "하지 말 것" 목록
2. **`PROGRESS.md` 생성** — 마일스톤별 체크리스트(아래 템플릿). 서브태스크 완료마다 체크하고 커밋.
3. **자주 커밋** — 작은 단위로. 커밋 메시지 마지막 줄에 `다음: <할 일>`.

**compact 발생/세션 복귀 시 복원 절차** (순서대로 읽고 이어서):
`CLAUDE.md` → 이 goal → spec → `PROGRESS.md` → `git log --oneline`

> context 60%에 근접하면, compact 전에 반드시 `PROGRESS.md`를 최신화하고 커밋해 둔다.

### PROGRESS.md 템플릿
```
# 진행 상황
현재 마일스톤: M0

## M0 doctor
- [ ] Trivy/osv/dep-scan/atom 설치 점검
- [ ] 런타임(Java21/Node/메모리) 점검
- [ ] 누락 가이드 출력
## M1 정확한 SCA  ... (이하 각 마일스톤 체크리스트)
```

## 핵심 원칙 (위반 금지 — CLAUDE.md에도 복사)

1. **정확도 우선** — FP를 줄이는 레버를 모든 것에 우선.
2. **결정적/LLM 경계** — 스캐너 실행·정규화·도달성은 결정적 코드. Claude는 증거 해석자일 뿐, 판정 oracle이 아니다.
3. **내부는 typed finding 모델, SARIF는 출력 전용** (SARIF를 내부 canonical로 쓰지 말 것 — spec 6절).
4. **자동 억제 금지** — 억제는 사람이 확정 (spec 12절).
5. **부분 실패는 정상** — 스캐너가 죽거나 도달성이 타임아웃해도 나머지 결과는 유효하게 보고(partial-result). "전부 성공" 위장 금지.
6. **구현 언어 Python**, 어댑터 인터페이스 추상화(새 스캐너 = 어댑터 1개 추가, 코어 무변경).

## 마일스톤 (순차 — 각 검증 게이트 통과 후 다음으로)

각 마일스톤 시작 전 **spec 16절 + 해당 절을 재독**한다. 게이트 미통과면 다음으로 넘어가지 않는다.

### M0 — doctor
스캐너(Trivy/osv/dep-scan/atom) + 런타임(Java 21/Node/가용 메모리) 설치·적합성 점검 CLI. 누락 시 설치 가이드.
- **게이트**: 환경을 정확히 진단하고 누락을 안내.

### M1 — 정확한 SCA (★이 도구의 정확도 증명 단계)
Trivy+osv 어댑터(병렬·status·자원거버넌스) → typed 모델 정규화(골든 테스트) → 합의 dedup → **dep-scan 도달성(4중 안전장치: 크기인지 자동/캐싱/예산+폴백/doctor 점검, spec 10.1)** → typed findings + SARIF + Markdown. 취약 픽스처(도달 가능/불가 케이스) 제작.
- **게이트**: 도달 불가 취약점이 `reachability=unreachable`로 표기 + 도달성 on/off 비교로 **FP 감소를 수치 실측** + 예산 초과 시 `reachability=unknown` 폴백 + 골든·부분실패 테스트 green.

### M2 — Secret
Gitleaks 어댑터 + opt-in TruffleHog 유효성 검증(기본 off, `--verify-secrets`/`--network-off`/never-validate 정책, spec 8절).
- **게이트**: 시크릿 탐지·검증·network 정책 동작.

### M3 — SAST
Semgrep CE 어댑터. taint는 intraprocedural 한계임을 인지하고 보고서에 명시(spec 10.2).
- **게이트**: 취약 패턴 탐지, typed 모델 통합.

### M4 — 억제
억제 안전장치(provenance/evidence/expiry/scope) + baseline + OpenVEX 루프 + 의존성/콜패스 변경 시 stale 무효화(spec 12절). Claude는 후보 제안만, 사람이 확정.
- **게이트**: 억제 후 재발 억제 + 버전 변경 시 무효화 동작.

### M5 — deep (experimental)
SpotBugs+FindSecBugs 바이트코드 어댑터(빌드 연동). 빌드 실패 시 partial로 격리.
- **게이트**: 바이트코드 탐지, 빌드 실패 안전 처리.
